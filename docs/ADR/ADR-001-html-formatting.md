# ADR-001: HTML форматирование в Telegram-боте

## Статус
Accepted

## Контекст

Telegram Bot API поддерживает два режима форматирования: MarkdownV2 и HTML.
Изначально бот использовал MarkdownV2 с функцией `escapeMarkdown()` для экранирования.

Проблема: `escapeMarkdown()` экранировала точку как `\.`, что Telegram MarkdownV2
отвергал с ошибкой:
```
send error: Bad Request: can't parse entities
```

Это блокировало отправку сообщений с точками в тексте (имена секретов, curl-команды).

## Решение

Полный переход на HTML форматирование:
- `ParseMode = "HTML"` вместо `ParseMode = "MarkdownV2"`
- `escapeHTML()` вместо `escapeMarkdown()` (экранирует `&`, `<`, `>`)
- Примеры кода в `<pre>` вместо обратных кавычек
- Жирный текст в `<b>` вместо `*`

## Последствия

**Положительные:**
- Telegram корректно парсит HTML (точки, скобки, спецсимволы)
- Меньше проблем с экранированием
- Читаемый код форматирования

**Отрицательные:**
- HTML менее читаемый чем Markdown в raw-виде
- Нужно экранировать 3 символа вместо 16 (MarkdownV2)

**Тесты обновлены:**
- `TestEscapeMarkdown` → `TestEscapeHTML`
- `\(2\)` → `(2)`, `\(1\)` → `(1)` в проверках
