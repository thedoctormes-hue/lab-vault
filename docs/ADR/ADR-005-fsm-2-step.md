# ADR-005: 2-шаговый FSM для создания секретов

## Статус
Accepted

## Контекст

Создание секрета требует двух действий: ввод имени и ввод значения.
Нужен механизм сохранения состояния между сообщениями пользователя.

## Решение

Конечный автомат (FSM) с двумя состояниями:
```
waiting_name → waiting_value → done (auto token generation)
```

Состояние хранится в `map[int64]*session` (ключ — ChatID):
```go
type session struct {
    state  string
    name   string
    value  string
}
```

Сброс состояния: `/start`, `/cancel`, или после завершения создания.

## Последствия

**Положительные:**
- Пошаговый диалог — пользователь не путается
- Валидация на каждом шаге (пустое имя, дубликат)
- Автоматическая генерация токена после шага 2

**Отрицательные:**
- Состояние в RAM — при рестарте бота сбрасывается
- Нет таймаута на сессию (зависшая сессия остаётся навсегда)
