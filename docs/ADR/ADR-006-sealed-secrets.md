# ADR-006: Sealed Secrets — шифрование секретов в RAM

## Статус
Accepted

## Контекст

До v3.0.0 секреты хранились в RAM в открытом виде (`map[string]string`). При компрометации процесса (core dump, `/proc/<pid>/mem`, swap) все секреты утекают без возможности обнаружения.

Требования:
- Защита секретов в RAM без изменения API (обратная совместимость)
- Миграция существующих снимков без простоя
- Минимальные зависимости (только стандартная библиотека + golang.org/x/crypto)

## Решение

### Архитектура: два режима работы

```
VAULT_PASSWORD установлен → Sealed mode (шифрование ChaCha20-Poly1305)
VAULT_PASSWORD не задан  → Legacy mode (plaintext, как было)
```

**Sealed mode:**
1. При старте из `VAULT_PASSWORD` через Argon2id выводится `sealedKey` (32 байта, фиксированная соль)
2. Каждое значение шифруется: `sealValue(plaintext) → hex(nonce|ciphertext)`
3. При чтении расшифровывается: `unsealValue(sealed) → plaintext`
4. Снимок сохраняется как JSON (значения в hex), при загрузке — автоопределение формата

**Legacy mode:**
- Полная обратная совместимость: `Set`/`Get`/`List` работают как раньше
- Снимок сохраняется в старом формате (encryptSnapshot)

### Криптография

- **KDF:** Argon2id (time=1, memory=64MB, threads=4), фиксированная соль
- **AEAD:** ChaCha20-Poly1305 с случайным nonce (12 байт) для каждого значения
- **Формат:** `hex(nonce || ciphertext)` — самодостаточная строка, не требует внешнего хранения nonce

### Миграция снимков

При загрузке снимка в sealed mode:
1. Пробуем распарсить как JSON → успех → sealed формат
2. Ошибка → пробуем decryptSnapshot (legacy) → успех → перешифровываем через `Set`
3. Ошибка → снимок повреждён

Миграция прозрачная, без ручного вмешательства.

## Последствия

**Положительные:**
- Секреты в RAM зашифрованы — core dump / swap / ptrace не дают plaintext
- Обратная совместимость: legacy-снимки мигрируются автоматически
- Нулевые внешние зависимости (только `golang.org/x/crypto`, уже в go.mod)
- Два режима позволяют постепенный переход

**Отрицательные:**
- `VAULT_PASSWORD` всё ещё нужен при старте (но не хранится в памяти дольше необходимого)
- Производительность: каждое `Get` вызывает AEAD-расшифровку (незаметно при текущих объёмах)
- Фиксированная соль в Argon2id — при утечке `sealedKey` все значения расшифровываются (salt не добавляет энтропии к ключу, только замедляет brute-force пароля)

**Риски:**
- Если `VAULT_PASSWORD` слабый — Argon2id замедляет, но не предотвращает brute-force
- Секреты расшифровываются при чтении — окно утечки присутствует (но не постоянно, как в legacy)
