# ADR-003: Токены доступа к секретам с TTL

## Статус
Accepted

## Контекст

Лаборантам нужен доступ к секретам для деплоя и конфигурации.
Хранение секретов в коде или логах недопустимо.

## Решение

Система токенов доступа:
- Каждый секрет может иметь множество токенов
- Токен привязан к одному секрету (SecretName)
- TTL по умолчанию: 720 часов (30 дней)
- Токены хранятся в config.yaml с yaml-тегами
- Автоматическая генерация при создании секрета

```go
type SecretToken struct {
    SecretName string    `yaml:"secret_name"`
    Token      string    `yaml:"token"`
    CreatedAt  time.Time `yaml:"created_at"`
    ExpiresAt  time.Time `yaml:"expires_at"`
    Revoked    bool      `yaml:"revoked"`
}
```

## Последствия

**Положительные:**
- Blast radius ограничен: утечка токена = утечка одного секрета
- TTL ограничивает время жизни токена
- Возможность отзыва токенов

**Отрицательные:**
- Токены хранятся в config.yaml (plain text)
- One-time токены не инвалидируются после использования (известный баг)
- Нет rate limiter на `/access/:token`

**Известные проблемы:**
- Токены не работали после перезапуска из-за отсутствия yaml-тегов (исправлено)
